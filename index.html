1. What is Data Communication?

Data communication refers to the exchange of data between two devices through some form of transmission medium such as a wire cable or wireless channel.

2. What are the key components of a data communication system?

Message, Sender, Receiver, Transmission medium, Protocol

3. What are protocols in networking?

Protocols are a set of rules that govern data communication — specifying how data is transmitted, formatted, and received.

4. What is the difference between a Hub, Switch, and Router?

Hub: Broadcasts data to all ports (Layer 1)
Switch: Forwards data to specific MAC addresses (Layer 2)
Router: Forwards packets based on IP addresses (Layer 3)

5. What are the layers of the OSI model?

Physical, Data Link, Network, Transport, Session, Presentation, Application

6. Which layer handles routing?

The Network layer (Layer 3) handles routing and logical addressing (IP).

7. What is the main function of the Transport layer?

To provide reliable process-to-process data delivery (e.g., TCP, UDP).

8. What are the main types of transmission media?

Guided: Twisted pair, coaxial cable, fiber optic
Unguided: Radio, microwave, infrared

9. What is the difference between TCP and UDP?

TCP: Connection-oriented, Reliable, Slower (HTTP, FTP)
UDP: Connectionless, Unreliable, Faster (DNS, VoIP)

10. What is IP addressing?

IP addressing provides a unique logical address to each device on a network, enabling proper routing of packets.

11. What is the difference between IPv4 and IPv6?

IPv4: 32 bits, Decimal format (192.168.1.1), ~4 billion addresses
IPv6: 128 bits, Hexadecimal format (2001:db8::1), Almost unlimited addresses

12. What is a subnet mask?

A subnet mask divides the IP address into network and host portions, determining the size of the network.

13. What is a router?

A router is a device that connects multiple networks and forwards data packets based on IP addresses.

14. What is the purpose of the routing table?

It stores paths and rules for forwarding packets to the correct destination network.

15. What are static and dynamic routing?

Static Routing: Routes manually configured by the administrator.
Dynamic Routing: Routes learned automatically using routing protocols (e.g., RIP, OSPF, EIGRP).

16. What is RIP (Routing Information Protocol)?

A distance-vector routing protocol that uses hop count as the metric and sends updates every 30 seconds.

17. What is the maximum hop count allowed in RIP?

15 hops. A hop count of 16 means the destination is unreachable.

18. What is OSPF?

Open Shortest Path First — a link-state routing protocol that uses Dijkstra's algorithm to calculate the shortest path.

19. What are the differences between RIP and OSPF?

RIP: Distance Vector, Hop Count metric, Slow convergence, 15 hop limit
OSPF: Link State, Cost (Bandwidth) metric, Fast convergence, No hop limit

20. What is a default route?

A route used by the router when no other route matches the destination IP address.

21. What is a gateway of last resort?

The router used as a last option to forward packets when no specific route is known — often configured as a default route.

22. What is meant by "ping" in networking?

ping checks network connectivity by sending ICMP Echo Request packets and waiting for Echo Reply.

23. What is ARP?

Address Resolution Protocol maps an IP address to its physical MAC address.

24. What is DNS?

Domain Name System translates human-readable domain names (e.g., google.com) into IP addresses.

25. What is DHCP?

Dynamic Host Configuration Protocol automatically assigns IP addresses to devices on a network.

26. What is NAT?

Network Address Translation converts private IP addresses to a public IP for internet communication.

27. What is QoS in networking?

Quality of Service ensures priority for important traffic (like voice or video) to reduce latency and packet loss.

28. What is the difference between unicast, multicast, and broadcast?

Unicast: One-to-one communication
Multicast: One-to-many selected nodes
Broadcast: One-to-all in the network

29. What is congestion in a network?

When the load on the network exceeds its capacity, leading to delay and packet loss.

30. What is meant by network topology?

The physical or logical arrangement of network devices — e.g., Bus, Star, Ring, Mesh, Hybrid.

31. What is a Mesh Topology?

Every device is connected to every other device for high redundancy.

32. What is a Star Topology?

All devices connect to a central switch or hub.

33. What is CSMA/CD?

Carrier Sense Multiple Access with Collision Detection — used in Ethernet.

34. What is CSMA/CA?

Carrier Sense Multiple Access with Collision Avoidance — used in Wi-Fi.

35. What is EIGRP?

Enhanced Interior Gateway Routing Protocol — a hybrid routing protocol by Cisco.

36. What is Static Routing?

Manually configured routing paths in a router.

37. What is Dynamic Routing?

Automatically updates routes using protocols like RIP or OSPF.

38. What is a VLAN?

Virtual LAN — logically divides a network into separate broadcast domains.

39. What is the function of the Session Layer?

Manages and controls dialogues between computers (session setup, maintenance).

40. What is the Presentation Layer responsible for?

Data formatting, encryption, and compression.

41. What is the Application Layer responsible for?

Provides services like email (SMTP), file transfer (FTP), and browsing (HTTP).

42. What is a Socket?

An endpoint for communication between two devices over a network.

43. What is a Port Number?

A numerical identifier for a specific process or service (e.g., 80 for HTTP).

44. What is ICMP?

Internet Control Message Protocol — used for network diagnostics (Ping, Traceroute).

45. What is SMTP?

Simple Mail Transfer Protocol — used for sending emails between mail servers.

46. What is POP3 and IMAP?

POP3: Downloads mail to a local system.
IMAP: Manages mail directly on the server.

47. What is FTP and TFTP?

FTP: File Transfer Protocol — uses TCP (port 21).
TFTP: Trivial File Transfer Protocol — uses UDP (port 69).

48. What is Bandwidth?

The data transfer capacity of a communication channel, measured in bits per second (bps).

49. What is Latency?

The delay between sending and receiving data over a network.

50. What is Throughput?

The actual amount of data successfully transmitted per unit time.

51. What is a Firewall?

A security device that monitors and controls incoming and outgoing network traffic based on predetermined security rules.

52. What is the difference between Synchronous and Asynchronous Transmission?

Synchronous: Data is sent in blocks with synchronization.
Asynchronous: Data is sent byte by byte with start and stop bits.

53. What is a Bridge?

A device that connects two LAN segments and forwards frames based on MAC addresses, operating at the Data Link layer.

54. What is a Gateway?

A device that connects networks using different protocols and can translate between them.

55. What is HTTP and HTTPS?

HTTP: HyperText Transfer Protocol (port 80)
HTTPS: HTTP Secure with SSL/TLS encryption (port 443)

56. What is SSL/TLS?

Secure Sockets Layer/Transport Layer Security — protocols that provide encryption and authentication for secure communication.

57. What is a Proxy Server?

An intermediary server that acts as a gateway between clients and other servers, providing caching and security features.

58. What is Load Balancing?

Distributing network traffic across multiple servers to ensure optimal resource utilization and prevent overload.

59. What is a DMZ?

Demilitarized Zone — a network segment that sits between the internal network and external network, providing an additional layer of security.

60. What is VPN?

Virtual Private Network — creates a secure, encrypted connection over a public network like the internet.

61. What is the difference between LAN, MAN, and WAN?

LAN: Local Area Network (small area)
MAN: Metropolitan Area Network (city-wide)
WAN: Wide Area Network (large geographical area)

62. What is Ethernet?

A family of networking technologies for LANs, defining wiring and signaling standards for the Physical and Data Link layers.

63. What is Wi-Fi?

Wireless Fidelity — a wireless networking technology that uses radio waves to provide network connectivity.

64. What is Bluetooth?

A short-range wireless communication technology for connecting devices over short distances (typically 10 meters).

65. What is Frame Relay?

A packet-switching protocol for connecting devices on a WAN, operating at the Data Link layer.

66. What is ATM?

Asynchronous Transfer Mode — a switching technique that uses fixed-length cells for data transmission.

67. What is MPLS?

Multiprotocol Label Switching — a routing technique that uses labels to make data forwarding decisions.

68. What is BGP?

Border Gateway Protocol — an exterior gateway protocol used to exchange routing information between autonomous systems on the internet.

69. What is SNMP?

Simple Network Management Protocol — used for monitoring and managing network devices.

70. What is Telnet and SSH?

Telnet: Unencrypted remote login protocol (port 23)
SSH: Secure Shell — encrypted remote login protocol (port 22)

71. What is a Collision Domain?

A network segment where data packets can collide with one another when being sent on a shared medium.

72. What is a Broadcast Domain?

A logical division of a network where all nodes can reach each other by broadcast at the data link layer.

73. What is Traceroute?

A diagnostic tool to show the route packets take from source to destination and measure transit delays.

74. What is Network Segmentation?

Dividing a network into smaller segments to improve performance, security, and manageability.

75. What is Packet Switching vs Circuit Switching?

Packet Switching: Data divided into packets, each routed independently
Circuit Switching: Dedicated communication path established for entire session

76. What is the difference between Ping and Traceroute?

Ping checks connectivity between two devices using ICMP Echo messages.
Traceroute shows the path (hops) that packets take to reach the destination.

77. How does network congestion affect performance?

Congestion causes packet loss, delay, and retransmissions. Can be simulated by adding traffic generators or limiting link bandwidth.

78. What is the role of ICMP in Ping?

Ping uses ICMP Echo Request and Echo Reply messages to test reachability. Routers and hosts process and reply to these packets.

79. How do you interpret Traceroute output?

Each line shows a router hop with delay time. High latency or timeouts indicate congestion or routing issues at that hop.

80. How can packet loss due to congestion be prevented?

Use QoS (Quality of Service), traffic shaping, congestion avoidance algorithms (RED), or increase bandwidth.

81. What is the purpose of assigning IP addresses in a LAN?

To uniquely identify devices in the LAN for data communication and routing.

82. What does the 'show mac address-table' command display?

Displays the MAC addresses learned by the switch and their associated ports.

83. What is the purpose of subnetting in IPv4?

Divides a large network into smaller subnets for efficient IP management and security.

84. How do you assign IPv6 address manually?

In Packet Tracer: interface g0/0 → ipv6 address 2001:db8::1/64

85. Why is a router required to connect LANs?

Routers connect different subnets and perform IP-based packet forwarding between networks.

86. What is the significance of gateway IP?

It acts as the exit point for traffic leaving the local subnet to reach other networks.

87. How does TCP ensure reliability?

Through acknowledgments, sequence numbers, and retransmissions. UDP sends datagrams without checking delivery.

88. What applications are best suited for TCP vs UDP?

TCP: Web, Email, File transfer
UDP: VoIP, Online games, Video streaming

89. What is the difference between Distance Vector and Link State routing?

Distance Vector (RIP): Shares entire routing table with neighbors
Link State (OSPF): Shares only link info, builds full topology

90. Why is OSPF better than RIP?

OSPF converges faster, supports large networks, uses cost metric instead of hop count.

91. What is the purpose of Wireshark in network analysis?

To capture, inspect, and analyze network packets for troubleshooting and study.

92. How do you apply filters in Wireshark?

Examples: HTTP → http, ICMP → icmp, TCP → tcp.port == 80

93. What is the difference between Datagram and Stream sockets?

Datagram = UDP (no connection). Stream = TCP (connection-oriented).

94. What is the role of sendto() and recvfrom() functions?

sendto() sends data to destination address; recvfrom() receives incoming datagrams.

95. What are essential devices for wireless network simulation?

Wireless router, Access point, Wireless PCs, and sometimes a server.

96. What is the role of SSID and WPA2 in wireless networks?

SSID identifies the network; WPA2 secures it using encryption and authentication.

97. What are the email protocols and their ports?

SMTP: Port 25 (sending)
POP3: Port 110 (receiving)
IMAP: Port 143 (receiving)

98. What is the importance of DNS in email systems?

Converts domain names (e.g., mail.local) into IP addresses for routing emails.

99. What factors influence routing protocol convergence?

Network size, topology change frequency, update intervals, protocol type.

100. What are the limitations of wireless networks?

Signal interference, lower speed, limited range, and potential security risks.
